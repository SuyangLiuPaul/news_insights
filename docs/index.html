01 Overview

News Insights is a dedicated reader for a bilingual news feed that pairs each headline — World, China, or Australia — with a Bible verse an AI has matched to it, plus a short reflection connecting the two. It doesn't run its own scraping or AI pipeline: all of that lives upstream in yswords-data, a shared data source originally built for a sister Bible-reading app and kept CORS-enabled specifically so other clients like this one could read it.

This project is the client only — a Flutter app (web, Android, iOS, macOS from one codebase) that fetches, caches, and renders that feed.

02 Architecture & data flow

Everything upstream of this app is someone else's problem on purpose — the app's job starts at a single JSON URL.

Why a separate data pipeline?

The verse-pairing feature originally lived inside the main YsWords Bible-reading app, but was removed there since it had no clear owner inside a Bible-reading app's scope. The pipeline itself kept running — it's genuinely useful on its own — so this app was built to give it a dedicated home rather than reinventing the AI-matching and translation logic.

03 Data schema

The feed is a single JSON document per edition (the live one at data/daily_news.json, or an archived day at data/archive/YYYY-MM-DD.json), organized as sections.{world,china,australia}.items[]. Every text field ships in both languages up front — there is no per-locale endpoint.

04 Client-side data strategy

The app has no backend of its own. RemoteDataService<T> is a small generic base class every data-backed screen builds on, with a three-tier fallback so the feed always has something to render immediately:

  1. SharedPreferences cache — the last successful network response, read synchronously on startup.
  2. Bundled asset — a snapshot shipped inside the app itself, used only when there's no cache yet (a brand-new install).
  3. Network fetch — always attempted in the background regardless of which of the above rendered first; on success it updates both the in-memory bundle and the persisted cache.

A background refresh that silently updates its own cache is only half the job — a long-lived screen also needs to be told when fresher data has arrived, or it can be stuck rendering its very first snapshot indefinitely. RemoteDataService exposes a broadcast updates stream for exactly this: FeedPage subscribes once and re-renders whenever a background refresh actually swaps in newer content, instead of requiring a manual pull-to-refresh to ever see it.

05 Engineering notes

A few fixes worth documenting because the root cause wasn't where it first looked.

Rendering

Cold-load CJK text race

Flutter Web's CanvasKit renderer doesn't bundle Chinese glyphs — the browser resolves a system fallback font asynchronously, and that resolution didn't always retrigger a relayout, so a cold load straight into Chinese could show garbled text until some unrelated state change (like toggling EN/中文) forced a redraw. Fixed by bundling Noto Sans SC as a real Flutter asset font, making CJK glyph availability part of the same guaranteed-ready-before-first-paint pipeline as the icon fonts.

Networking

A third of all images silently failing

SBS News images return Access-Control-Allow-Origin: null — an explicit CORS deny, not just a missing header — which blocks CanvasKit from decoding them, even though the exact same URL loads fine via a plain server-side fetch (which doesn't enforce CORS). Guardian, BBC, and DW all send a permissive ACAO: *; only SBS didn't. Fixed upstream in yswords-data by mirroring SBS images into its own CDN with an open CORS header — the same pattern already used there for another image set.

Caching

Fixes that didn't reach already-affected browsers

Three compounding gaps meant a shipped fix didn't self-propagate: the bundled fallback snapshot hadn't been regenerated since the very first commit (still carrying pre-fix image URLs), the background refresh had no way to tell the UI a fresher bundle had arrived (see above), and an already-cached stale snapshot had no self-healing path. Fixed all three: regenerated the bundled asset, added the update stream, and bumped the cache key to force one clean refetch for every existing install.

i18n

"Matthew 5:9" instead of "马太福音 5:9"

The verse reference field is English-only upstream — there's no referenceZh, unlike every other text field. lib/utils/book_name_localizer.dart splits a reference into book name + chapter:verse and maps the book name through a Simplified-Chinese table (ported from the main YsWords app's book-name mapping), covering all 66 books plus the corpus's singular "Psalm" form — with a graceful fallback to the original string for anything unrecognized.
